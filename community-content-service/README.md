# community-content-service

**Документация Swagger** - `/community-content/api/docs`
Сервис пользовательского контента: закладки, лайки и рецензии к фильмам.

Хранит данные в отдельной схеме `community_content` в общей PostgreSQL.

Консистентность пользовательских данных (лайки, рецензии) и справочников (пользователи, фильмы) сейчас никак не обеспечивается т.к. в этом нет необходимости. Это намеренное решение для полной изоляции сервисов. Полное удаление записей или пользователей не предполагается.

## Эндпоинты

Все эндпоинты расположены под префиксом `/community-content/api/v1` и требуют JWT.

- **Закладки** (`/bookmarks`): добавить, удалить, получить список.
- **Лайки** (`/likes`): выставить/обновить/удалить оценку фильма, получить статистику фильма.
- **Рецензии** (`/reviews`): создать, удалить, получить список с сортировкой,
  проголосовать (лайк/дизлайк) за рецензию.

## Миграции

Новые миграции генерируются так же, как в `auth-service`:

```bash
cd community-content-service
poetry install
source .venv/bin/activate
alembic revision --autogenerate -m "<description>"
```
